# Invariants（不變式總表）

> 自動閘的脊椎。每條不變式都應有對應的 property/unit test；debug build 每 tick assert，release build 關閉。
> 違反不變式的 bug 一律以 `seed + tick 區間 + input trace` 回報，並指出壞掉的 tick。

## drug-graph（分子 / 多圖效果系統）

- **變換正確性**：translate 掃動 + 牆停 + 危險即死；scale 以固定有理比例往各圖原點拉；swap 交換兩圖位置。
- **朝向正確性**：旋轉 4 次 = 原狀；flip 兩次 = 原狀；旋轉/flip 對平移類變換施加正確。
- **療效判定可重現**：逐圖「最終位置」判定 deterministic、可重現。
- **重排不變**：保持每台機器朝向 + 單顆處理順序 → 各圖最終結果不變（與 belt 佈局無關）。
- **防抄性質**：多數藥方下，把所有平移步驟同步 +90° 會改變結果（藍圖無法無腦旋轉復用）。

## mapgen + solver

- **建構即可解**：生成的地圖至少存在那條被構造出來的多圖解。
- **生成確定性**：同 seed → 逐欄位相等的地圖（含每圖原點）+ 相同難度分。
- **難度界限**：難度分落在設定區間內（不無腦、不需荒謬機器數）。
- **定價一致**：同藥方 → 同基礎藥價。
- **求解器健全（soundness）**：回傳非 null 的解，確實治到目標且全程未失敗。
- **跨圖張力（cross-map tension）**：每張生成的關卡至少有一種疾病的標準解必須「解耦」各圖——即包含 swap / scale / reverse / perpendicular / offset 之一。建構手法（對 N ∈ {2,3,4} 張圖一致）：各圖給「不同原點」（N ≤ 4 時每圖原點皆相異，是解耦的前提；否則同步平移下各圖位置永遠相等），疾病以 `d % nMaps` 輪流落在各自的圖上，並在每種解藥座標 (cx,cy) 於「其餘所有圖」放置 Hazard——天真的同步前推（朝目標直推、所有圖一起動）會把藥掃進那個 Hazard 而報廢，最佳解因此被迫解耦（`scale` 把每圖各自拉向自己的原點，故解耦在任意 N 都成立，不倚賴只連 0/1 兩圖的 swap01）。生成端以求解器 oracle 驗證此性質後才接受。求解器是 (W·H)^N 的 BFS，故 N 越大地圖要越小：N=2 可到 ~16×16（預設/CLI）、N=3 用 ~7×7、N=4 用 ~6×6，搭配溫和難度區間。

## factory-sim

- **質量守恆**：每 tick，流入 = 流出 + 在途 + 庫存增量。
- **不憑空生滅**：無物品憑空生成或消失；merge/split 不複製、不吞物品。
- **死鎖可偵測**：有界 buffer 下能偵測並標記死鎖 tick（不靜默卡死）。
- **吞吐一致性**：穩態產出速率 = 瓶頸機器速率。
- **確定性**：同 build + seed + input → replay 兩次 hash 相同。

## economy

- **帳務守恆**：現金變動 = 收入 − 支出。
- **庫存非負**。
- **反退化**：規則確實讓單品收益遞減（狂產單一藥物 ≠ 簡單最佳解）。

## save

- **round-trip**：`deserialize(serialize(state))` 深等於 `state`（多存檔皆成立）。
